docker-compose.yml 文件中的配置，可以定义多个服务、网络、卷和其他配置选项。下面是一些常见的配置选项：

version：定义 Compose 文件格式的版本。
services：定义应用程序中的服务，每个服务可以由一个或多个容器组成。
build：定义 Docker 镜像的构建方式，可以是使用 Dockerfile 构建，也可以是使用外部构建上下文构建。
image：指定容器的镜像名称，可以是公共 Docker Hub 中的镜像，也可以是本地的自定义镜像。
ports：将容器的端口映射到主机的端口，使得外部用户可以通过主机访问容器的服务。
volumes：定义容器的卷，用于持久化数据或共享数据。
environment：定义容器的环境变量，用于设置容器内的应用程序配置。
depends_on：定义服务之间的依赖关系，确保在启动一个服务之前，所依赖的服务已经启动。
networks：定义 Docker 网络，用于服务之间的通信。
restart：定义容器的重启策略，在容器崩溃或停止时自动重启容器。
command：指定容器启动时要执行的命令或脚本。覆盖默认容器启动命令
healthcheck：定义容器的健康检查方式，可以检查应用程序是否正常运行。
deploy：用于定义 Docker Swarm 的部署选项，例如副本数量、部署策略等。
